# Codex

## Команды

| Команда               | Описание                                                  |
|-----------------------|-----------------------------------------------------------|
| `codex`               | Режим: локальная интерактивная работа                     |
| `codex cloud КОМАНДА` | Команды для работы с облачными задачами                   |
| `codex resume --all`  | Список всех интерактивных сессий                          |
| `codex login`         | Вход в Codex через ChatGPT OAuth, device auth или API key |
| `codex logout`        | Выход из Codex и удаление сохраненных учетных данных      |

https://developers.openai.com/codex/cli/reference#command-overview

## Список /statusline в Codex

| Пункт                  | Что показывает                            | Зачем нужен                                                                             |
|------------------------|-------------------------------------------|-----------------------------------------------------------------------------------------|
| `context-usage`        | Индикатор использования контекста         | Помогает понимать, когда старая история может начать сжиматься или учитываться хуже     |
| `five-hour-limit`      | Использование лимита за последние 5 часов | Помогает понимать, сколько лимита уже израсходовано и как скоро он начнет освобождаться |
| `weekly-limit`         | Остаток недельного лимита использования   | Помогает понимать, сколько лимита уже израсходовано и как скоро он начнет освобождаться |
| `git-branch`           | Текущая git-ветка                         | Быстро видно, в какой ветке идет работа                                                 |
| `model-name`           | Только имя модели, без reasoning          | Более короткий вариант, чем model-with-reasoning                                        |
| `model-with-reasoning` | Текущая модель вместе с уровнем reasoning | Самый информативный вариант отображения модели                                          |
| `project-root`         | Корень проекта                            | Полезно, если запуск не из корня репозитория                                            |
| `current-dir`          | Текущая рабочая директория                | Удобно, если работаешь в нескольких папках                                              |

Если совсем просто, то `five-hour-limit` зависит от:

- твоего тарифа;
- того, работаешь ли ты локально или через cloud task;
- сложности и длины задач;
- объема контекста, который Codex вынужден держать.

## Установка

1. [Настроить терминал](../windows/windows-terminal/README.md)
2. [Установить/переустановить Ubuntu на wsl](../windows/wsl.md)

```shell
sudo apt update

# Node.js + npm
sudo apt install nodejs npm -y

# codex
sudo npm install -g @openai/codex
```

Назначить прокси:

```shell
nano ~/.bashrc

# Добавить переменные
export HTTP_PROXY=http://xx.xx.xx.xx:3128
export HTTPS_PROXY=http://xx.xx.xx.xx:3128

# Применить переменные без перезапуска
source ~/.bashrc
```

При первом запуске надо залогиниться в ChatGPT.  
Codex откроет ссылку в браузере по умолчанию. Если браузер тоже работает через прокси, авторизация должна пройти нормально.
