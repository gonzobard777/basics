# auto-in

Поиск и просмотр истории команд bash через **curses-меню на стрелке ↑**.   
Замена тяжёлым инструментам вроде Atuin/fzf: без базы данных, без внешних
зависимостей (нужен только `python3`, модуль `curses` входит в стандартную
поставку).

## Файлы

```
install.sh      установщик (клей для bash вшит внутрь)
uninstall.sh    деинсталлятор (обратная операция)
auto-in.py      движок-меню (Python + curses), копируется в ~/.local/bin/
```

Установщик раскладывает:

- `~/.local/bin/auto-in.py` — движок (копия `auto-in.py`);
- `~/.hist_search.bash` — клей (связующий bash-слой) для readline;
- строку подключения клея в `~/.bashrc` (+ `~/.local/bin` в `PATH`).

## Как устроено

**Движок (`auto-in.py`)** — читает историю из stdin (записи разделены NUL),
рисует на `/dev/tty` интерактивное меню с live-фильтром, печатает выбранную
команду в stdout. Код возврата сообщает, что нажато: `0` — Enter (выполнить),
`3` — Tab (подставить), `1` — Esc (отмена). Многострочные команды показываются
одной строкой (переносы — как `⏎`), но подставляются/выполняются целиком. Если
терминала нет (`/dev/tty`/`TERM` недоступны) — тихо выходит с кодом 1, без
traceback.

**Клей (`~/.hist_search.bash`)** — *glue code*, связующий bash-слой между readline
и движком. Сам по себе ни bash, ни Python-движок не знают друг о друге; клей их
«склеивает» (своей логики поиска в нём нет — она вся в `auto-in.py`):

1. вешает на ↑ readline-макрос (`\C-xU\C-xE`);
2. собирает историю (`__hist_list`) и скармливает движку;
3. ловит вывод движка и кладёт в строку (`READLINE_LINE`);
4. по коду возврата движка решает — выполнить (`accept-line`) или просто подставить;
5. плюс вся обвязка общей истории между вкладками (`history -n/-s/-a`, DEBUG-trap)
   и зависимости (`histappend`, `ignoredups`).

Подробности механизма:

- **Двухшаговый макрос.** ↑ разворачивается в `\C-xU` (запустить движок, подставить
  выбранное) + `\C-xE` (шаг «акцепт»). `bind -x` сам не умеет «нажать Enter» за
  readline, поэтому функция по коду возврата движка делает `\C-xE` либо
  `accept-line` (выполнить команду), либо no-op (только подставить).
- **Оба keymap'а.** Привязки вешаются и в `emacs-standard`, и в `vi-insert`, так
  что ↑ работает в том числе при `set -o vi`, не только в режиме по умолчанию.
- **Запрос через `--`.** Движку передаётся `... -- "$READLINE_LINE"`, чтобы текст
  вроде `--test` искался как подстрока, а не воспринимался как опция движка.

## Установка

```bash
bash install.sh
exec bash          # подхватить изменения в текущем терминале
```

Установщик **идемпотентный** — запускать можно сколько угодно раз (файлы
обновятся, строки в `~/.bashrc` не продублируются).

## Удаление

```bash
bash uninstall.sh
exec bash
```

Убирает движок, клей и подключение из `~/.bashrc` (перед правкой делает бэкап
`~/.bashrc.auto-in.bak`; удаляются только две точные строки установщика, ваши
строки не трогаются). Строку `PATH` не трогает (ею могут пользоваться другие
программы из `~/.local/bin`). Тоже идемпотентно.

## Использование

Нажмите **↑** в пустой/заполненной командной строке:

| Ситуация      | Что покажет меню                         |
|---------------|------------------------------------------|
| строка пустая | всю историю, свежие сверху               |
| набран текст  | только команды, содержащие эту подстроку |

Внутри меню (видно 10 строк, дальше — прокрутка):

| Клавиша   | Действие                                      |
|-----------|-----------------------------------------------|
| ↑ / ↓     | навигация / прокрутка (также Ctrl-P / Ctrl-N) |
| печать    | сужает фильтр (подстрока, регистронезависимо) |
| Backspace | стереть последний символ фильтра              |
| **Enter** | выбрать и **сразу выполнить**                 |
| **Tab**   | подставить в строку **без запуска**           |
| **Esc**   | отмена                                        |
| Ctrl-U    | очистить запрос                               |

Фильтр в меню принимает печатные **ASCII**-символы; не-ASCII (например кириллицу),
набранную в самом меню, движок игнорирует. Чтобы сузить по такому тексту, наберите
его в командной строке **до** нажатия ↑ — он станет начальным фильтром.

## Общая история между вкладками

Команды видны в других вкладках сразу — **и выбранные из меню, и набранные
вручную**, в том числе долгоживущие (`claude`, `ssh`, `npm install …`), ещё пока
они выполняются. Достигается это записью в `~/.bash_history` в момент **до**
запуска команды, двумя путями:

- **Выбор из меню (Enter):** `__on_up` сразу делает `history -s` + `history -a`.
- **Ручной ввод:** DEBUG-trap (`__auto_in_flush` → `history -a`) дописывает
  набранную команду перед её выполнением.

Обычный bash так не умеет: историю он сохраняет только при выходе из шелла, а
типичный приём с `PROMPT_COMMAND` срабатывает лишь когда вернётся приглашение —
то есть уже **после** завершения команды. Долгоживущий процесс держит шелл
занятым, приглашение не возвращается, и его команда в другие вкладки не попадает.
Запись в момент ввода/выбора (а не по приглашению) закрывает этот разрыв.

Перед показом меню `history -n` подтягивает команды, которые успели дописать
другие вкладки.

Нюанс: уже **открытые** соседние вкладки увидят новую команду не мгновенно, а
при их следующем нажатии ↑ (тогда сработает `history -n`). **Новая** вкладка
видит всё сразу при старте.

### Зависимости (проставляются автоматически)

Фича опирается на два параметра, и клей выставляет их сам при подключении —
полагаться на содержимое `~/.bashrc` не нужно, а ваши настройки не затираются:

- `shopt -s histappend` — дописывать историю при выходе, а не перезатирать
  (иначе выход одной вкладки затёр бы команды других);
- `HISTCONTROL` с `ignoredups` — чтобы выбранная команда не задвоилась: её
  добавляет и `history -s`, и сам `accept-line`; `ignoredups` гасит дубль. Если
  в `HISTCONTROL` уже есть `ignoredups`/`ignoreboth`, клей ничего не меняет;
  иначе аккуратно добавляет `ignoredups` к текущему значению.

Кроме того, клей ставит **DEBUG-trap** (для записи набранных вручную команд) —
один раз на сессию (guard от повторного `source`). С `atuin`/`bash-preexec` это
совместимо: они ставят свой DEBUG-trap отложенно (на первом приглашении) и
**цепляют** уже существующий, поэтому подхватят наш — в каком бы порядке вы их
ни подключили в `~/.bashrc`. (Прочитать чужой trap прямо из клея нельзя: внутри
`source` DEBUG-trap для `trap -p` скрыт — это ограничение bash.)

При удалении (`uninstall.sh`) эти настройки откатываются сами собой — они живут
внутри клея, а он удаляется; пользовательский `HISTCONTROL` в `~/.bashrc` не
трогается. (DEBUG-trap, выставленный в уже работающих вкладках, исчезнет при их
закрытии; новые вкладки его не получат.)

## Зачем отдельный файл движка

Правится один файл `auto-in.py`, а `install.sh` разносит его при установке.
Файл `auto-in.py` должен лежать **рядом** с установщиком — иначе `install.sh`
сообщит об ошибке и не тронет текущую установку.
