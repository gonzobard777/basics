# auto-in

Поиск и просмотр истории команд bash через **curses-меню на стрелке ↑**.
Замена тяжёлым инструментам вроде Atuin/fzf: без базы данных, без внешних
зависимостей (нужен только `python3`, модуль `curses` входит в стандартную
поставку).

## Установка

```bash
bash auto-in.sh
exec bash          # подхватить изменения в текущем терминале
```

Установщик **идемпотентный** — запускать можно сколько угодно раз (файлы
обновятся, строки в `~/.bashrc` не продублируются).

## Использование

Нажмите **↑** в пустой/заполненной командной строке:

| Ситуация      | Что покажет меню                         |
|---------------|------------------------------------------|
| строка пустая | всю историю, свежие сверху               |
| набран текст  | только команды, содержащие эту подстроку |

Внутри меню (видно 10 строк, дальше — прокрутка):

| Клавиша   | Действие                                      |
|-----------|-----------------------------------------------|
| ↑ / ↓     | навигация / прокрутка                         |
| печать    | сужает фильтр (подстрока, регистронезависимо) |
| **Enter** | выбрать и **сразу выполнить**                 |
| **Tab**   | подставить в строку **без запуска**           |
| **Esc**   | отмена                                        |
| Ctrl-U    | очистить запрос                               |

## Файлы

```
auto-in.sh      установщик (клей для bash вшит внутрь)
hist-menu.py    движок-меню (Python + curses), копируется в ~/.local/bin/
```

Установщик раскладывает:

- `~/.local/bin/hist-menu.py` — движок (копия `hist-menu.py`);
- `~/.hist_search.bash` — клей для readline;
- строку подключения клея в `~/.bashrc` (+ `~/.local/bin` в `PATH`).

## Как устроено

**Движок (`hist-menu.py`)** — читает историю из stdin, рисует на `/dev/tty`
интерактивное меню с live-фильтром, печатает выбранную команду в stdout.
Код возврата сообщает, что нажато: `0` — Enter (выполнить), `3` — Tab
(подставить), `1` — Esc (отмена).

**Клей (`~/.hist_search.bash`)** — вешает на ↑ readline-макрос из двух шагов:
`\C-xU` (запустить движок и подставить выбранное в строку) + `\C-xE` (шаг
«акцепт»). `bind -x` сам не умеет «нажать Enter» за readline, поэтому функция
по коду возврата движка включает `\C-xE` либо в `accept-line` (выполнить
команду), либо в no-op (только подставить).

## Зачем отдельный файл движка

Правится один файл `hist-menu.py`, а `auto-in.sh` разносит его при установке.
Файл `hist-menu.py` должен лежать **рядом** с установщиком — иначе `auto-in.sh`
сообщит об ошибке и не тронет текущую установку.
